Socket.io

From the official webstie: Socket.io enables real-time, bidirectional and event-based communication between clients and servers. It works on every platform, browser or device, focusing equally on reliability and speed.

Socket.io is a JavaScript library for realtime web applications. It has a client side library that runs in the browser and a server side library for Node.js. Socket.io primairly uses the WebSocket protocol for client/server communications. However, polling is used as a fallback option as well.

Set Up Instructions:

1. Make sure node is installed and you have done an npm init on your current project directory (see the Node page for more details).
2. Use the node package manager (npm) to install socket.io on the server: -> on the command line in your project working directory run the command "npm install --save socket.io-client"
3. Client: include "/socket.io/socket.io.js" as a script in the html file whose elements cause client server communication.
4. Click the Socket.io logo at the top of the page for specific instructions on using socket.io as well as combining it with:
* A node http server
* Express

Notes:

- Socket.io provides two way client server communication.
- Socket.io is relatively simple to understand and get started with.
- To use Socket.io you must be running a Node.js server.
- Socket.io is built on asynchronous networking libraries. This causes server load and makes it a poor choice for applications with a large number of client connections.
